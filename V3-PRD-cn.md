# Chromium Web Summarizer CRX V3 构思（乔布斯视角）

## 0. 这版要解决的核心问题
当前产品已经可用，但仍偏“工具集合”：功能齐全，心智不够锋利。
V3 的目标不是再加按钮，而是把用户从“读完”直接推进到“做完”。

一句话定义：
- 它不是网页摘要器，它是「一页内容 -> 一周行动」的决策机器。

## 1. 乔布斯式产品主张（Product Thesis）

### 1.1 真正的北极星
- 北极星不是摘要质量分，而是 `Action Started Rate`（看完后 24 小时内至少完成 1 个步骤）。

### 1.2 三条铁律
- 聚焦：每次只回答 3 件事：值不值得做、为什么、下一步。
- 一体化：总结、判断、行动在同一屏完成，不跳转，不打断。
- 默认即高级：新手无需学习即可成功，高手不被阉割（保留 API/Prompt 预设）。

### 1.3 该砍掉什么
- 砍掉“功能存在感”，保留“结果存在感”。
- 不在主界面暴露参数思维（温度、Token、Endpoint），这些都留在设置页。
- 不追求“全能知识管理”，只做“当前页面的高质量决策”。

## 2. 用户体验重构：从功能流到决策流

### 2.1 首页只保留一个主动作
- 主按钮：`给我结论`
- 次动作：`换来源`、`换总结方式`
- 其他全部降噪（包括主题、历史管理等控制项弱化）

### 2.2 输出结构固定为“决策卡片”
输出永远按这个顺序呈现：
1. 一句话判断（Amazing / Shit + 结论）
2. 三条关键洞察（判断 + 解释）
3. 应该砍掉的部分（砍掉 + 解释）
4. 一周执行步骤（目标 / 步骤 / 预期）

原则：
- 先结论，后论证，最后行动。
- 用户 8 秒内必须看到“该不该做”。

### 2.3 执行优先模式（Focus Mode）升级
- 进入 Focus Mode 后默认只展示：
  - 一句话判断
  - 执行步骤清单
- 其他内容折叠到“查看推理”。

这是“把复杂留给机器，把方向留给人”。

## 3. 可信度设计：让用户知道什么时候该信、什么时候别信

### 3.1 新增「可信度条」
在结果顶部展示 3 个信号：
- 内容充分度（正文长度、有效段落）
- 噪音程度（重复率）
- 模型稳定度（是否结构完整）

输出文案示例：
- `可信度：高（正文充分，结构完整）`
- `可信度：中（正文偏短，建议人工复核）`

### 3.2 失败不只报错，要给下一步
每个错误都必须附可执行修复动作：
- `API 鉴权失败 -> 打开设置页检查 Key 与 Endpoint`
- `正文过短 -> 建议切换到文章正文页后重试`

## 4. 双层产品策略（保留现有能力）

### 4.1 默认层（90% 用户）
- 一个主按钮
- 两个可见切换（来源 / 总结方式）
- 自动质量提示
- 可直接勾选执行步骤

### 4.2 专业层（10% 用户）
- 完整保留：API 预设、Prompt 预设、模型拉取、参数调优
- 但入口收敛在 Options，不进入主工作流

结论：不是删功能，而是把复杂度“重新分层”。

## 5. 商业与增长的产品化种子（不影响当前单机形态）

### 5.1 可传播单位
新增一键导出：
- `决策卡片 Markdown`
- `执行清单纯文本`

让用户自然把结果贴到飞书/Notion/微信，不做社交功能，也能增长。

### 5.2 连续使用钩子
- 摘要历史中突出“未完成步骤数”
- 打开历史时优先显示“上次未完成”

目标：把“摘要工具”变成“每周复盘入口”。

## 6. 对当前代码的落地改造建议

### 6.1 `workspace.html / workspace.js`
- 主按钮文案从“总结当前来源网页”升级为“给我结论”。
- 将 `themeModeBtn`、`clearHistoryBtn` 降级到二级区域或折叠菜单。
- 默认首屏展示“判断 + 步骤”，洞察与砍掉项折叠。
- 为质量卡新增“可信度等级一句话”。

### 6.2 `background.js`
- 在 `meta.quality` 基础上增加 `confidenceLabel`（高/中/低）。
- 增加输出结构校验：缺失 `## 1)` 或 `## 4)` 时自动重试一次（轻量自愈）。

### 6.3 `options.js`
- 保持现有能力，但增加“推荐模板”标签：
  - 决策模式（默认）
  - 信息提取模式

## 7. 指标与验收（V3）

### 7.1 核心指标
- `TTFS <= 8s`（首个摘要时间）
- `Action Started Rate >= 45%`
- `Step Completion D1 >= 25%`（24 小时内至少勾选 1 步）
- `Summary Retry Rate <= 15%`（首次结果不满意导致重试）

### 7.2 体验验收
- 新用户首次打开后 30 秒内可完成一次总结并复制步骤。
- 用户无需进入设置页也能完整跑通主流程。
- 错误态均有明确下一步，不出现“只报错不指路”。

## 8. 90 天演进路线

### Phase 1（1-2 周）
- 改文案、改信息层级、强化 Focus Mode、可信度文案上线。

### Phase 2（3-5 周）
- 输出结构校验与自愈重试。
- 历史项增加“未完成步骤”可视化。

### Phase 3（6-12 周）
- 页面类型识别（新闻/长文/产品页）自动匹配 Prompt。
- Side Panel 形态验证（不替代现有工作台）。

## 9. 最后一句话（产品判断）
如果一个总结不能直接触发行动，它就只是信息压缩。
V3 要做的是：把“看懂”变成“开干”。
