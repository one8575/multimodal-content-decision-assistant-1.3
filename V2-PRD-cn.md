# Chromium Web Summarizer CRX V2 PRD（中文，兼容保留版）

## 1. 文档信息
- 产品名称：`chromium-web-summarizer-crx`
- 文档版本：`V2.0-compat`
- 文档日期：`2026-05-16`
- 文档定位：在保留当前版本功能与用户需求的前提下，升级为更快、更稳定、更可执行的网页总结产品

## 2. 背景与约束
- 当前版本已具备可用主链路：抓取网页 -> 调用 LLM -> 结构化总结 -> 历史与执行清单。
- 用户明确要求：暂不砍掉现有“可配置项”和“多模式能力”，本版必须兼容保留。
- 因此 V2 采用“双层产品策略”：
  - 默认层：给大多数用户更简单、更快的体验。
  - 专业层：完整保留 API/Prompt 预设、模式切换、参数调优能力。

## 3. 目标与非目标
## 3.1 目标
- 将“首次可用时间”缩短到 10 秒以内（首次打开后即可完成一次总结）。
- 将“总结到执行”的转化率提高（复制步骤、勾选步骤、二次查看历史）。
- 保留并强化现有高级配置能力，不破坏老用户工作流。

## 3.2 非目标
- 本阶段不做账号系统与云端多端同步。
- 本阶段不做复杂知识库与跨页面长期记忆。
- 本阶段不重构为聊天式 Agent 产品。

## 4. 北极星指标与成功标准
- NSM：`Actionable Summary Rate`（产生总结后触发“复制步骤/勾选步骤/再次打开历史”任一行为的占比）
- 关键指标：
  - `TTFS`（Time To First Summary）<= 10 秒
  - 总结成功率 >= 95%
  - 执行行为触发率 >= 40%
  - 7 日回访率 >= 25%

## 5. 用户与典型场景
## 5.1 用户分层
- 快速用户：只想一键得到结论和下一步。
- 专业用户：需要切换模型、端点、Prompt，并对输出结构可控。

## 5.2 核心场景
- 场景 A：读文章/报告时，快速提炼判断、洞察、执行步骤。
- 场景 B：同一页面用不同 Prompt 模式复盘（乔布斯模式 vs 通用提取模式）。
- 场景 C：沉淀历史摘要并在一周内跟踪执行清单。

## 6. V2 产品原则
- 默认可用：不用先学习配置也能跑通主流程。
- 结构刚性：输出必须包含判断、洞察、砍掉项、下一步。
- 兼容保留：高级能力不删除，只做入口分层和体验优化。
- 速度优先：优先优化抓取和请求稳定性，再扩展功能。

## 7. 信息架构（兼容版）
- 工作台（Workspace）
  - 主区：总结结果（Markdown/执行视图）
  - 执行板：步骤勾选、复制步骤
  - 历史区：最近摘要、快速回看
  - 顶部：总结按钮、来源切换、模式切换、设置入口
- 设置页（Options）
  - API 预设管理（保留）
  - Prompt 预设管理（保留）
  - 默认预设选择（保留）

## 8. 功能需求（V2）
## 8.1 总结主流程优化
- 点击“总结当前来源网页”后，状态分段可见：
  - 抓取中 -> 请求中 -> 规整中 -> 完成/失败
- 失败提示分层：
  - 页面内容不足
  - API Key 缺失
  - Endpoint/模型响应异常

## 8.2 输出质量与可执行性
- 继续沿用结构化输出模板，确保可解析步骤。
- 在结果区新增“质量提示”：
  - 提取内容长度
  - 疑似噪音过高提醒（如正文过短、重复度高）

## 8.3 历史与执行增强
- 历史项增加元信息展示：
  - 使用的 Prompt 名称
  - 抓取来源域名
  - 生成时间
- 执行清单保持本地持久化，支持历史项内状态恢复。

## 8.4 默认层与专业层分层
- 默认层（工作台）仅暴露必要操作：
  - 总结、复制、切换来源、切换模式
- 专业层（设置页）保留全部配置：
  - API 预设、Prompt 预设、参数调优、默认项切换

## 9. 明确保留项（基于用户要求）
- 不移除 API 预设功能。
- 不移除 Prompt 预设功能。
- 不移除“总结方式”切换。
- 不移除“只看执行模式”。
- 不移除历史摘要与执行清单。

## 10. 技术方案要点
- 继续使用 MV3 架构：
  - `background.js`：标签选择、脚本注入、API 请求、输出规整
  - `workspace.js`：交互渲染、历史与步骤状态
  - `options.js`：预设管理与参数保存
- 数据存储策略：
  - `chrome.storage.sync`：API/Prompt 预设与当前默认配置
  - `chrome.storage.local`：历史摘要、执行状态、UI 模式偏好
- 兼容性原则：
  - 新字段只增不减，旧字段迁移时保留 fallback 逻辑

## 11. 埋点与数据定义（本地事件模型）
- 事件建议：
  - `summary_start`
  - `summary_success`
  - `summary_fail`
  - `copy_markdown`
  - `copy_text`
  - `copy_steps`
  - `step_toggle`
  - `history_open`
- 关键属性：
  - `prompt_id`
  - `source_domain`
  - `content_length`
  - `duration_ms`
  - `error_type`（失败时）

## 12. 里程碑
- M1（1 周）：输出质量提示 + 状态分段 + 错误分层文案
- M2（1 周）：历史元信息增强 + 执行清单恢复稳定性
- M3（1 周）：默认层/专业层分层优化 + 指标回收校验

## 13. 验收标准（UAT）
- 新用户首次安装后，不进入设置也可完成总结（使用默认预设）。
- 老用户升级后，已有 API/Prompt 预设不丢失且可继续使用。
- 结构化输出可稳定解析出执行步骤，步骤状态刷新后不丢失。
- 常见错误可被清晰提示，用户知道下一步如何修复。

## 14. 风险与缓解
- 风险：网页正文提取质量不稳定。
  - 缓解：增加质量提示与短内容保护；后续引入更强提取策略。
- 风险：不同 OpenAI 兼容接口格式差异导致失败。
  - 缓解：保持 endpoint 归一化与详细错误透传。
- 风险：预设配置复杂导致新用户困惑。
  - 缓解：默认层弱化配置曝光，专业层承接高级能力。

## 15. 后续演进（V2.x）
- V2.1：Side Panel 形态验证（与独立工作台并行，不强制替换）。
- V2.2：页面类型识别策略（新闻、文档、博客）与 Prompt 自动匹配。
- V2.3：导出能力增强（Markdown 模板导出、任务卡片导出）。
