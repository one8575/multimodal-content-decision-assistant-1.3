# 多模态内容决策助理 - 新手使用说明

## 这是什么

**多模态内容决策助理** 是一个 Chrome/Edge 扩展，用来把网页内容、本地文本或粘贴内容快速整理为可执行结论。

它适合你在阅读大量信息后，快速得到：
- 一句话判断
- 关键洞察
- 该砍掉的部分
- 一周可执行步骤

## 你能用它做什么

- 总结网页文章、报告、长文
- 总结你粘贴的会议纪要、笔记
- 总结本地文本文件（如 `.txt`、`.md`、`.csv`、`.json`）
- 切换不同总结风格（Prompt 预设）
- 保存历史总结，回看过去结果

## 核心功能

- 输入来源三选一：当前网页 / 粘贴文本 / 上传文件
- 支持 OpenAI 兼容 API
- 可配置模型参数：Endpoint、API Key、Model、Temperature、Max Tokens
- 一键复制结果（Markdown、富文本、步骤）
- 历史记录管理与清空

## 第一次使用（5 分钟）

1. 打开浏览器扩展页：`chrome://extensions/`（或 `edge://extensions/`）
2. 开启“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择项目目录：`./multimodal-content-decision-assistant-1.3`
5. 安装后，点击扩展图标进入工作台

## 首次配置 API

1. 在工作台点击 `API 设置`
2. 填写：
   - Endpoint（如 OpenAI 兼容接口地址）
   - API Key
   - Model
   - Temperature（建议 0.2）
   - Max Tokens（建议 600 起）
3. 点击 `保存`

## 日常使用流程

1. 选择输入来源：网页 / 粘贴 / 文件
2. 选择“总结方式”（Prompt 预设）
3. （可选）填写补充上下文：目标、受众、约束、输出偏好
4. 点击 `生成结论`
5. 在“结论概览”和“完整结果”查看输出
6. 需要复用时，点击复制按钮

## 三种常见场景

### 场景 1：快速评估一篇文章
- 打开文章页
- 输入来源选“当前网页”
- 选择“乔布斯决策模式”
- 生成后直接看“一句话判断 + 三条洞察”

### 场景 2：整理会议纪要
- 输入来源选“粘贴文本”
- 粘贴纪要全文
- 在“目标”里写：提炼执行项
- 生成后优先看“一周执行清单”

### 场景 3：分析本地报告
- 输入来源选“上传文件”
- 上传 `.md/.txt/.csv/.json`
- 生成后复制 Markdown 给团队

## 提示：怎样让结果更好

- 输入正文尽量完整（建议 200 字以上）
- 在“补充上下文”里写清楚你的目的
- 先用低温度做稳定总结，再按需提高创造性
- 针对不同任务切换不同 Prompt 预设

## 隐私与数据说明

- API 配置与历史记录存储在浏览器本地（`chrome.storage`）
- 你可以在工作台中清空历史
- 发布代码到 GitHub 前，不会自动包含你浏览器里的历史和 API 配置

## 常见问题

### 1) 提示缺少 API Key
去 `API 设置` 页面填入并保存。

### 2) 结果为空或失败
检查：
- Endpoint 是否正确
- API Key 是否可用
- Model 是否存在
- 网络是否可访问该 API

### 3) 网页无法总结
可能网页正文太短、页面受限，或不是普通 http/https 页面。

---
如果你是第一次使用，建议先用一篇公开文章做测试，确认 API 可用后再导入你的实际内容。
